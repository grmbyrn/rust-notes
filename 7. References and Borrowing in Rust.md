# 🦀 Rust: Ownership, Functions & Borrowing — Key Concepts

## 🔑 Key Takeaways

### 1. **Ownership with Functions**

* Passing a variable to a function **transfers ownership** (unless the type implements `Copy`).
* Once ownership is moved, the original variable becomes invalid.
* To retain ownership, either:

  * Return the variable from the function, or
  * Pass it by **reference** (borrowing).

---

### 2. **Returning Ownership**

* Functions can return ownership of data.
* You can:

  * Give ownership to a function (by passing a value).
  * Take ownership from a function (via return values).
  * Use both (e.g., pass a value in and get it back out).

---

### 3. **Borrowing with References**

* References allow you to **use a value without taking ownership**.
* Syntax: `&var` (immutable reference), `&mut var` (mutable reference).
* Borrowing is called **"creating a reference"**.

---

### 4. **Mutable vs Immutable References**

* Rule 1: You can have **either**:

  * One mutable reference, **or**
  * Any number of immutable references,
  * **But not both at the same time in the same scope**.

---

### 5. **Dangling References**

* A **dangling reference** points to memory that is no longer valid.
* Rust **prevents this at compile time** by refusing to compile code that returns references to data that will be dropped.

---

### 6. **Reference Rules Recap**

* ✅ At any given time:

  * You may have **one mutable reference** OR **many immutable references**.
* ❌ You cannot have both in the same scope.
* ✅ References must always be valid (no dangling pointers).

---

## ✅ Rust MCQs: Ownership, Borrowing, and References

> Try answering before expanding the correct answer and explanation.

---

### 1. What happens when you pass a `String` to a function?

A) The value is copied
B) The value is cloned
C) Ownership is transferred to the function
D) Nothing happens

<details>
<summary>✅ **Reveal Answer**</summary>

**Correct Answer:** C
**Explanation:** Passing a `String` to a function **moves ownership**. After this, the original variable cannot be used unless the value is returned.

</details>

---

### 2. What is the best way to **use** a variable in a function without giving up ownership?

A) Clone the variable
B) Use a reference (`&`)
C) Use a pointer
D) Use a global variable

<details>
<summary>✅ **Reveal Answer**</summary>

**Correct Answer:** B
**Explanation:** Borrowing via references (`&var`) lets you use a value in a function without transferring ownership.

</details>

---

### 3. Why does this Rust code fail?

```rust
let mut s = String::from("hello");
let r1 = &s;
let r2 = &mut s;
```

A) It uses too much memory
B) It has more than one mutable reference
C) It mixes mutable and immutable references in the same scope
D) The compiler doesn't support strings

<details>
<summary>✅ **Reveal Answer**</summary>

**Correct Answer:** C
**Explanation:** You can't mix mutable and immutable references in the **same scope**. This prevents potential data races at compile time.

</details>

---

### 4. What does the Rust compiler do if you try to return a reference to a local variable?

A) Compiles but gives a warning
B) Returns a null reference
C) Errors at runtime
D) Errors at compile time

<details>
<summary>✅ **Reveal Answer**</summary>

**Correct Answer:** D
**Explanation:** Rust detects **dangling references** at **compile time**, preventing returning references to memory that will be dropped.

</details>

---

### 5. Which of the following is allowed by Rust’s borrowing rules?

A) Multiple mutable references to a variable
B) A mutable reference and immutable reference in same scope
C) One mutable reference to a variable
D) Returning a reference to a local variable

<details>
<summary>✅ **Reveal Answer**</summary>

**Correct Answer:** C
**Explanation:** Rust allows **only one mutable reference** at a time, and it must not coexist with immutable references in the same scope.

</details>

---

### 6. What is the term for using a reference to a value instead of taking ownership?

A) Cloning
B) Dereferencing
C) Borrowing
D) Copying

<details>
<summary>✅ **Reveal Answer**</summary>

**Correct Answer:** C
**Explanation:** This is called **borrowing** in Rust. It allows functions to read or modify data without taking ownership.

</details>
